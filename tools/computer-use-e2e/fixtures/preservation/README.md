# Preservation Fixture

This fixture powers `tools/computer-use-e2e/preservation-harness.mjs`.

It is a sanitized hybrid fixture:

- `state.seed.json` is derived from a known-good Product Proof run with real
  machine, path, PR, git, process, event, and env-key details normalized.
- `screenshots/` contains only the curated real screenshots required by current
  visual contracts. The full historical artifact tree is intentionally not
  checked in.
- Text snapshots are generated by the harness at runtime from the labels in the
  seed state.
- Expected JSON files are normalized signatures, not full raw HTML snapshots.

Acceptance runs must use:

```bash
node tools/computer-use-e2e/preservation-harness.mjs run
```

`update-fixtures --allow-rewrite` is only for intentional report/state contract
changes. Review the generated diffs before committing them.
