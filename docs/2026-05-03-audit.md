**Findings**

1. ✅ `apps/native/src/tauri-api.ts:241` calls `git_init_if_needed`, but Rust exposes/registers `git_init_repo` instead (`apps/native/src-tauri/src/commands.rs:282`, `apps/native/src-tauri/src/main.rs:332`). If `initIfNeeded()` is used, it will fail at runtime.

1. The Rust contract is split across multiple modules and only partially generated. Specta covers `shared_types.rs` and `sqlite_types.rs`, but command-facing types in `types.rs`, `permissions.rs`, `scanner.rs`, `peek.rs`, `editor.rs`, `finalize_apply.rs`, and `commands.rs::CommitResult` are manually mirrored in `tauri-api.ts`.

1. Several Rust commands return `serde_json::Value` instead of typed structs, which weakens the contract: examples include `config_set_host_attr`, `git_init_repo`, `git_stash`, `darwin_evolve_cancel`, `nix_check`, `ui_set_prefs`, `apply_system_defaults`, and `darwin_build_check`.

1. `sqlite_types.rs` uses raw `String` for constrained values like `ChangeSummary.status`, `QueuedSummary.status`, and `QueuedSummary.summary_type` (`apps/native/src-tauri/src/sqlite_types.rs:56`, `68`). These should be enums at the domain/API layer, even if DB rows stay string-backed.

1. `ApplyResult` is overloaded. `types::ApplyResult` means process output (`ok/code/stdout/stderr`), while `finalize_apply::ApplyResult` means `{ git_status, evolve_state }`. The frontend `ApplyResult` matches the second meaning, not the first.

1. Frontend types are manually broader or different than Rust in places. `config.get` is typed as `DarwinConfig | null`, but Rust returns non-null `Config`; `git.status` is typed nullable, but Rust returns non-null or error; `FeedbackMetadata` has `lastPromptText` in TS but not Rust.

**Removal**

- Removed the protobuf scaffold.
- No `proto/` files remain.
- Current git status only shows pre-existing untracked `.claude/` and `mockups/`.

**Recommendation**
Make Rust the source of truth, but consolidate it:

1. Create a `contract` or `api_types` Rust module for every Tauri command input/output/event type.
1. Derive `Serialize`, `Deserialize`, and `specta::Type` for all command-facing structs/enums.
1. Replace ad hoc `serde_json::Value` returns with small typed structs like `OkResponse`, `NixCheckResult`, `BuildCheckResult`, and `ConfigEditApplyResult`.
1. Generate the TS API types from Rust, then remove duplicated hand-written interfaces from `tauri-api.ts`.
1. Optionally use `tauri-specta` to generate typed command bindings instead of manually writing `invoke<T>()` wrappers.
